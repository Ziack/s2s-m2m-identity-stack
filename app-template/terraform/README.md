# Terraform root for this service

This directory is generated by `npm create @s2s/service@latest`.

## Prerequisites

1. The `s2s-platform` Terraform has already been applied in the target AWS account, and its outputs are published to SSM under `/<environment>/s2s/platform/*`.
2. AWS credentials with at least:
   - `ssm:GetParameter` on `/<environment>/s2s/platform/*`
   - Permissions documented in the `s2s-service` module README

## Usage

```bash
terraform init
terraform apply \
  -var image_tag=<your-image-tag> \
  -var alb_listener_rule_priority=<unique-priority>
```

Re-apply after every image push:

```bash
terraform apply -var image_tag=<new-tag> -var alb_listener_rule_priority=<same>
```

## What this calls

A single `module "s2s_service"` reference pinned to `?ref=v<sdkVersion>` of the published modules. Inputs are derived from prompt answers given to the CLI at scaffold time. To change the bounded context, scopes, or path pattern after scaffolding, edit `main.tf` directly.
